Phase 1: 기본 골격 및 디자인 테마 설정 

프로젝트 시작: "결혼식 모바일 초대장 프로젝트를 시작할게. 기술 스택은 HTML5 하나에 모든 걸 담고, 스타일링은 Tailwind CSS CDN을 사용해줘. 아이콘은 FontAwesome CDN을 써줘." 

폰트 설정: "폰트는 구글 폰트의 'Noto Sans KR'(본문용)과 'Noto Serif KR'(제목용)을 가져와서 CSS 변수나 클래스로 설정해줘." 

헤더 디자인: "화면 상단에 꽉 차는 배경 이미지를 넣고, 그 위에 '지훈 & 수진', 날짜, 장소 텍스트를 흰색으로 중앙 정렬해서 올려줘. 배경 이미지는 약간 어둡게 처리해서 글씨가 잘 보이게 해줘." 

D-Day 기능: "헤더 중앙에 결혼식 날짜(2025년 12월 25일)를 기준으로 D-Day를 계산해서 보여주는 텍스트 영역을 만들어줘. 스크립트는 나중에 짤 거니까 일단 HTML 구조만 잡아줘." 

메인 컨테이너: "메인 콘텐츠가 헤더 이미지 위로 살짝 겹쳐서 올라오는 카드 형태의 디자인을 잡아줘. 배경은 흰색이고 모서리는 둥글게." 

PWA 설치 배너 UI: "메인 상단에 '앱으로 보기'라는 PWA 설치 유도 배너를 만들어줘. 평소엔 숨겨져 있다가(hidden), 나중에 조건이 맞으면 보이게 할 거야. 문구는 **'앱으로 저장하고 편리하게 이용하세요.'**로 수정해줘."

초대 문구 섹션: "감성적인 결혼 초대 문구가 들어갈 섹션을 만들어줘. 문구 아래에는 신랑, 신부의 이름과 전화 걸기 아이콘을 배치해줘." 

갤러리 UI: "2열 그리드(Grid) 형태로 사진 4장을 보여주는 갤러리 섹션을 만들어줘. 이미지는 placeholder로 넣어줘." 

섹션 위치 수정: "스마트 편의 기능 섹션을 초대 문구 섹션 바로 아래, 갤러리 섹션 이전에 배치해줘."

Phase 2: 핵심 기능 UI 구현 

APK 다운로드 UI: "PWA 배너 밑에 'Android 앱 (풀 기능 버전) 다운로드'라는 별도의 카드 섹션을 만들고, 구글 드라이브 다이렉트 다운로드 URL에 연결되는 버튼을 배치해줘. 버튼 클래스는 bg-blue-500을 사용해줘." 

오시는 길(지도) UI: "오시는 길 섹션을 만들 건데, 텍스트 주소 밑에 약도 이미지(map.png)가 들어갈 영역을 꽉 차게 잡아줘." 

지도 버튼: "지도 이미지 바로 아래에 네이버 지도, 카카오 지도, 구글 지도로 연결되는 버튼 3개를 가로로 배치해줘. 각 브랜드 색상(초록, 노랑, 회색)을 맞춰줘." 

기능 버튼 그리드: "알림 받기 버튼과 식권 QR 스캔 버튼을 2열 그리드로 배치해줘. 알림 버튼의 텍스트를 '알림 받기', 식권 버튼의 텍스트를 '식권 QR 스캔'으로 해주고, 아이콘을 크게 넣고 설명을 작게 덧붙여줘. 알림 버튼의 상태를 나타낼 수 있는 data-state 속성도 추가해줘." 

푸터: "맨 아래에 저작권 표시(Copyright)를 작게 넣어줘." 

Cordova 스크립트 추가: "HTML 바디 맨 끝에 cordova.js를 로드하는 스크립트 태그를 추가해줘." 

Phase 3: 로직 구현 - 하이브리드 앱 처리 

기본 스크립트 시작: "이제 <script> 태그 안에 자바스크립트를 짤 거야. 먼저 결혼식 날짜를 상수로 정의하고, D-Day를 계산해서 화면에 업데이트하는 함수를 만들어줘." 

Cordova 감지: "현재 앱이 웹 브라우저에서 실행 중인지, Cordova 앱 내에서 실행 중인지 감지하는 변수 isCordova를 정의해줘 (!!window.cordova 사용)." 

PWA 배너 표시 로직 수정: "beforeinstallprompt 이벤트를 감지하는 로직을 만들되, isCordova가 아닐 때(웹일 때)는 이벤트 발생 유무와 관계없이 '앱으로 설치하기' 배너가 무조건 표시되도록 로직을 수정해줘."

설치 버튼 클릭 핸들러: "설치 버튼을 누르면 설치 프롬프트를 띄우는 비동기 함수 installPWA를 구현해줘." 

지도 링크 로직: "지도 버튼을 눌렀을 때 이동하는 openMap(type) 함수를 만들어줘. 네이버, 카카오, 구글 각각의 URL scheme이나 웹 링크로 새 창(_system)을 띄우도록 해줘." 

알림 버튼 상태 함수: "알림 버튼의 텍스트와 스타일을 '알림 받기'/'알림 취소' 상태에 따라 업데이트하고 예약된 시간을 표시하는 updateNotificationButton 함수를 구현해줘." 


알림 기능 로직 (Date Picker 적용): "toggleNotification 함수를 만들 건데, isCordova가 참일 때만 작동하도록 분기해줘. cordova-plugin-datepicker 플러그인을 사용하여 사용자가 원하는 날짜와 시간을 선택하게 하고, cordova-plugin-local-notification을 사용하여 그 시간에 알림이 예약되도록 로직을 짜줘. 이미 알림이 예약되어 있다면 취소하는 기능도 포함해줘." 

QR 스캔 기능: "scanQR 함수를 만들어줘. 이건 isCordova일 때만 작동해야 해. navigator.camera 플러그인을 사용해서 사진을 찍는(스캔 흉내) 로직을 넣고, 웹이면 '앱에서만 가능합니다'라고 경고창을 띄워줘." 

Cordova Ready: "deviceready 이벤트 리스너를 추가하여 알림 권한 요청 및 초기 알림 상태를 확인하고 updateNotificationButton 함수를 호출하는 로직을 넣어줘." 

스크립트 통합: "작성된 모든 자바스크립트 로직을 index.html의 하단 스크립트 영역에 합쳐줘." 

Phase 4: PWA 필수 파일 생성 

Manifest 파일 생성: "이 웹사이트를 PWA로 설치할 수 있게 manifest.json 파일을 만들어줘. 이름은 '지훈&수진 결혼식', 테마 컬러는 핑크색, 아이콘 경로는 새로운 봉투 이미지 URL로 넣어줘." 

Manifest 연결: "index.html의 <head> 태그 안에 manifest.json을 연결하고, iOS용 메타 태그(apple-touch-icon의 아이콘 경로도 새로운 봉투 이미지 URL로 수정하여) 추가해줘." 

Service Worker 전략: "오프라인에서도 사이트가 열리도록 service-worker.js 파일을 만들 거야. 캐시 이름은 'wedding-pwa-v1'으로 하고, index.html, manifest.json 및 새로운 봉투 아이콘 이미지를 포함한 CDN 리소스들을 캐싱해줘." 

캐싱 로직: "서비스 워커에 'Install', 'Activate'(구 버전 캐시 삭제), 'Fetch'(네트워크보다 캐시 우선 사용) 이벤트를 구현해줘." 

Service Worker 등록: "index.html의 스크립트에서 페이지 로드 시 서비스 워커를 등록하는 코드를 추가해줘." 

이미지 경로 수정: "아까 만든 HTML에서 갤러리 이미지 경로를 img1.jpg 등으로, 지도 이미지는 map.png로 확실하게 지정해줘." 

Phase 5: 최종 검토 및 파일 정리 

전체 코드 리뷰: "index.html 파일 전체를 다시 한번 정리해서 보여줘. 중복된 부분 없이 깔끔하게 합쳐줘." 

파일 분리: "manifest.json과 service-worker.js 코드도 각각 별도의 코드 블록으로 명확하게 구분해서 다시 보여줘." 

마무리: "이 코드를 Cordova 프로젝트의 www 폴더에 넣고 빌드하기 위해 추가로 설치해야 할 플러그인 명령어와 config.xml에 명시해야 할 POST_NOTIFICATIONS 권한을 알려줘."
